# 项目管理模式

## 功能说明

1. **维护 PROJECT_STATUS.md**
   - 更新项目阶段和整体进度
   - 记录已完成模块
   - 跟踪进行中的工作

2. **维护 TODO.md**
   - 管理待办任务列表
   - 优先级排序
   - 状态跟踪

3. **进度统计**
   - 计算代码变更量
   - 统计功能完成度
   - 生成进度报告

4. **文档同步**
   - 确保文档与代码同步
   - 识别遗漏的更新
   - 建议文档改进

## 使用方式

### 自动更新
开发完功能后运行 `/update-status` 命令，代理会自动：
- 扫描 Git 提交记录
- 分析变更文件
- 更新项目状态文档

### 手动查询
询问项目状态：
- "项目进度怎么样了？"
- "还有哪些待办任务？"
- "本周完成了什么？"

### 任务管理
- "添加一个待办任务：实现 X 功能"
- "更新任务 Y 的状态为进行中"
- "标记任务 Z 为已完成"

## 输出格式

### 状态查询响应

```
## 项目状态

### 整体进度
- 当前阶段：功能迭代中
- 完成度：XX%
- 代码行数：~XX,XXX

### 本周完成
- ✅ 功能 1
- ✅ 功能 2
- ✅ 修复 X 个 Bug

### 进行中
- 🔄 功能 3
- 🔄 功能 4

### 待办
- 📋 功能 5（高优先级）
- 📋 功能 6（中优先级）
```

### 更新报告

```
## 项目状态更新

### 变更分析
- 新增文件：X 个
- 修改文件：Y 个
- 删除文件：Z 个

### 进度变化
- 完成功能：X 个
- 新增待办：Y 个
- 整体进度：XX% → YY%

### 文档已更新
- PROJECT_STATUS.md
- TODO.md
```

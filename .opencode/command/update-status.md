# 更新项目状态

## 任务说明
扫描最近 Git 提交，分析代码变更，更新以下文档：
1. `docs/PROJECT_STATUS.md` - 项目状态
2. `docs/TODO.md` - 待办清单

## 执行步骤

### 1. 分析 Git 变更
- 获取最近 10 条提交记录
- 识别新增、修改、删除的文件
- 分类变更类型（功能/修复/重构/文档）

### 2. 更新 PROJECT_STATUS.md
- 更新"最后更新"日期
- 修改相关模块的完成状态
- 添加新完成的里程碑
- 更新整体进度百分比

### 3. 更新 TODO.md
- 将完成的任务从待办移到已完成
- 添加新的待办任务
- 更新优先级

### 4. 生成更新报告

## 输出格式

```
📝 项目状态更新报告

## 本次更新内容
- ✅ 新增已完成任务：X 项
- 📋 新增待办任务：Y 项

## 主要变更
1. [模块名]：变更描述
2. [模块名]：变更描述

## 进度变化
- 整体进度：XX% → YY%
- 新增完成：Z 个模块/功能

## 下一步建议
1. 建议操作 1
2. 建议操作 2
```

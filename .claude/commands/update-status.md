# /update-status - 自动更新项目状态

作为项目状态管理员，请自动更新 docs/PROJECT_STATUS.md 和 docs/TODO.md。

## ⚠️ 重要说明

此命令需要 Git，用于：
- 检测哪些文件被修改
- 关联提交记录与完成的任务
- 记录变更历史

## Git 不可用时：
此命令无法执行，建议以下替代方案：
1. 运行 `/progress` 了解当前状态
2. 手动编辑 docs/PROJECT_STATUS.md

## 你需要做的（Git 可用时）：

1. 读取现有 docs/PROJECT_STATUS.md
2. 查看最近 10 条 Git 提交
3. 扫描代码变化
4. 智能更新文档：
   - 将完成的任务移到"已完成"
   - 更新"进行中"任务的进度
   - 添加新发现的待办任务
   - 更新时间戳

## 输出格式：

# 📝 项目状态更新报告

## 本次更新内容
- ✅ 新增已完成任务：X 项
- 🚧 更新进行中任务：X 项
- 📋 新增待办任务：X 项

## 主要变更
1. 模块A：从进行中 → 已完成
2. 模块B：进度更新 XX%

## 📊 进度变化
- 整体进度：XX% → **XX%**

---
✅ 文档已更新：`docs/PROJECT_STATUS.md`
✅ 文档已更新：`docs/TODO.md`

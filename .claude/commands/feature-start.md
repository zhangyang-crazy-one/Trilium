# /feature-start - 开始新功能开发

作为功能开发助手，使用 feature-dev 插件进行结构化功能开发。

## 使用方式

```
/feature-start [功能描述]
```

示例：
```
/feature-start 实现文件搜索功能
/feature-start 添加 AI 对话历史导出
/feature-start 开发 Markdown 导入功能
```

## 执行流程

1. **需求理解**
   - 分析功能描述
   - 识别关键功能点
   - 确定验收标准

2. **创建开发计划**
   - 分解为子任务
   - 确定依赖关系
   - 估算复杂度

3. **调用 @feature-developer**
   - 使用 feature-dev 插件流程
   - 生成详细实施计划

4. **增量式开发**
   - 按照计划逐步实现
   - 每个步骤添加测试
   - 验证功能完整性

## 输出格式

```markdown
## 🚀 功能开发计划

### 需求理解
[对功能的理解]

### 任务分解
1. [任务 1]
2. [任务 2]
3. [任务 3]

### 实施步骤

#### 步骤 1: [名称]
- **描述**: XXX
- **代码变更**: XXX
- **测试要求**: XXX
- **验证方式**: XXX

#### 步骤 2: [名称]
...

### 进度跟踪
- [ ] 任务 1
- [ ] 任务 2
- [ ] 任务 3

### 后续操作
1. 按照步骤顺序开发
2. 每个步骤完成后调用 `/next` 获取下一步
3. 完成所有步骤后运行 `/update-status` 更新进度
```

## 与其他命令配合

- `/next` - 获取下一步开发建议
- `/progress` - 查看整体项目进度
- `/update-status` - 更新功能完成状态

## 注意事项

- 功能描述要具体明确
- 识别功能边界和范围
- 遵循增量式开发原则
- 及时更新文档和测试
